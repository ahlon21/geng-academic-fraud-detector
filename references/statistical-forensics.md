# 统计取证手册（第二式 / 第四式深化）

> 数据造假最致命的弱点：**编数据的人通常不懂统计的内在约束**。本手册给出一套
> 可以"用数学证伪"的检验，配合 `scripts/forensics.py` 直接出结论。

判定原则：**这些检验只能"证伪"，不能"证真"**。通过检验 ≠ 数据真实；
未通过检验 = 在数学上不可能，是硬证据。报告中务必区分这两种力度。

---

## 1. GRIM 检验（均值粒度一致性）

**适用场景**：因变量是**整数**（李克特量表、计数、整数评分、人数），且报告了
均值与样本量。

**原理**：n 个整数之和必为整数，故均值只能是 `整数 / n`。给定报告均值的小数位数，
若不存在任何整数和能修约成该均值，则该均值在数学上不可能。

**怎么用**：
```bash
python3 scripts/forensics.py grim --mean 5.19 --n 28
# 量表均值（k 个条目求平均）用 --items k
python3 scripts/forensics.py grim --mean 3.27 --n 15 --items 10
```

**判别力边界**：当 `n × items ≥ 10^小数位` 时，GRIM 几乎总通过、无意义。
所以 GRIM 对**小样本 + 多位小数**最有效（如 n=20、报告到两位小数）。脚本会自动提示。

**经典实锤**：mean=5.19, n=28 → 5.19×28=145.32，最近整数和 145→5.18、146→5.21，
无法得到 5.19。这是 Brown & Heathers (2017) 的原始案例。

---

## 2. GRIMMER 检验（均值+标准差联合一致性）

**适用场景**：整数数据，同时报告了均值与标准差（SD）。

**原理**：在 GRIM 基础上，平方和 `Σx²` 必为整数，且满足奇偶约束
`Σx² ≡ Σx (mod 2)`（因为对任意整数 `x² ≡ x (mod 2)`）。由报告 SD 的修约区间
反推 `Σx²` 的可行区间，若区间内不存在满足奇偶性的整数，则 (均值, SD) 组合不可能。

**怎么用**：
```bash
python3 scripts/forensics.py grimmer --mean 3.45 --sd 1.12 --n 20
```

GRIMMER 比 GRIM 更敏感——很多能通过 GRIM 的均值会被 SD 卡住。
对一张表里所有 `(mean, sd, n)` 行批量跑：
```bash
python3 scripts/forensics.py scan-table --file table.csv   # 需含 mean,n 列；可选 sd,items
```

**局限**：这是"分析型 GRIMMER"（Allard 2018），检查整数解的存在性与奇偶性，
但不含取值上下界可行性（那需要 SPRITE）。因此它能证伪，通过不代表一定真实。

---

## 3. SPRITE（样本重建，需人工/外部工具）

当 GRIMMER 通过但你仍怀疑时，可用 SPRITE（Heathers et al.）：给定
均值、SD、n、取值上下界，反向枚举**能产生该统计量的整数样本**。
若重建出的样本分布明显荒谬（如必须有大量极端值、双峰），即为危险信号。
本工具箱不内置 SPRITE 枚举（组合爆炸），建议用 R 包 `rsprite2` 或在线 SPRITE。
在报告中标注"建议 SPRITE 复核"。

---

## 4. Benford 首位数定律

**适用场景**：一大批**跨多个数量级、无人为上下限**的自然数字（如各种测量值、
金额、计数汇总）。**不适用**于受限量表、百分比、单一量级、刻意设定范围的数据。

**原理**：自然数据首位为 d 的概率 ≈ `log10(1 + 1/d)`（1 占 30.1%，9 仅 4.6%）。

```bash
python3 scripts/extract_pdf.py numbers paper.pdf -o nums.txt
python3 scripts/forensics.py benford --file nums.txt
```

读 MAD（平均绝对偏差，Nigrini 首位阈值）：
`<0.006 紧密符合 / 0.006–0.012 可接受 / 0.012–0.015 勉强 / >0.015 不符合`。
**注意**：n<50 不可靠；Benford 偏离≠造假，只是"值得深挖"的线索，不能单独定罪。

---

## 5. 末位数字均匀性

**适用场景**：一批测量值。真实测量的**最后一位**应近似均匀（0–9 各 ~10%）。

```bash
python3 scripts/forensics.py digits --file nums.txt
```

**红旗**：末位 0/5 严重超额（编数据爱凑整）、某些末位系统性缺失、
卡方 p<0.05。同样只是线索。

---

## 6. 基线/分布均匀性（Carlisle 法，人工）

在随机对照试验里，基线变量（年龄、身高、入组指标）的组间差异 **p 值**
应近似均匀分布在 [0,1]。若大量基线 p 值异常地靠近 1（组间"过于平衡"），
说明数据可能被编造或拼凑——这正是揭穿藤井善隆（183 篇撤稿）、Boldt 的方法。
做法：收集所有基线比较的 p 值，检验其分布是否偏离均匀（过度集中于高 p 值区）。

## 7. 方差/标准差异常

- **SD 过于整齐**：所有组 SD 都是整数或同一小数位 → 可疑。
- **SD 过小**：独立生物重复几乎不可能给出极小的组内方差。
- **跨组 SD 雷同**：不同处理组 SD 位数完全一致，像是套模板。
- **均值±SD 反推不自洽**：用 GRIM/GRIMMER 直接卡。

## 8. p 值与效应量自洽

- **p-hacking**：大量 p 值恰好落在 0.04–0.05 区间。
- **检验统计量与自由度不匹配**：报告的 t/F 值反推 p 与所述不符（可手算复核）。
- **小样本+超强显著**：n 很小却 p<0.001 且效应量巨大，需警惕。
- **选择性报告**：只报显著结果，方法里提到的指标在结果里消失。

---

### 工作流建议
1. `extract_pdf.py numbers` 抽出全部数字 → `benford` + `digits` 先扫一遍。
2. 把正文/表格里所有 `(mean, sd, n)` 整理成 CSV → `scan-table` 批量跑。
3. 单个可疑统计量 → `grim` / `grimmer` 精确判定。
4. RCT/有基线表 → 收集基线 p 值做 Carlisle 检验（人工）。
5. 任何"数学上不可能"的命中都是 🔴 级硬证据，写进报告时附上脚本输出。
